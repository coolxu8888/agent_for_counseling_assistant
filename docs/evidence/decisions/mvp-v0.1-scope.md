# 咨询师助理 Agent MVP v0.1 范围

## 1. MVP 定位

MVP v0.1 是咨询师助理 Agent 的第一阶段版本，目标是先支持咨询师在初访与单次 session 记录中的高频工作。

本阶段不追求完整治疗方案设计，也不替代咨询师进行诊断、风险处置或最终临床判断。Agent 的核心作用是：根据咨询师输入的信息，生成一套流派无关的基础信息收集、个案信息整理与 session 记录系统。

项目中可加入危机等级训练/评测数据，用于模型学习和测试“候选等级”判断。但 v0.1 仍不把候选等级作为真实个案的最终风险分级；真实个案必须由咨询师结合面询、机构流程和必要转介进行判断。

在 MVP v0.1 中，CPS/中国本地伦理与法律资料主要作为伦理、风险、记录规范和专业边界的底层参考，而不是把三个 P0 能力做成某一理论流派版本。境外专业资料可以作为补充参考，但必须标注地域限制。CBT、人本、精神动力、家庭系统等理论框架暂不进入 v0.1 的信息抓取模板，后续会在“个案概念化”和“咨询方案设计”等功能中按咨询师偏好调用。

## 2. 第一阶段优先能力

MVP v0.1 先实现三个能力：

| 优先级 | 能力 | 目标 |
|---|---|---|
| P0 | 初访信息收集表生成 | 帮助咨询师快速生成统一的初访访谈表、来访者预填写表或系统字段表 |
| P0 | 个案信息整理 | 将咨询师提供的个案背景、访谈笔记或零散记录整理为结构化摘要 |
| P0 | Session 总结与咨询记录生成 | 根据咨询师记录生成本次 session 总结、咨询记录和下次重点 |

## 3. 理论框架与 RAG 使用原则

本阶段三个能力应使用统一模板，不按不同咨询流派生成不同版本。无论咨询师后续使用 CBT、人本、精神动力、家庭系统或整合取向，初访信息收集、个案信息整理和 session 记录都应先完成一套基础、稳定、可复用的信息抓取。

CPS/中国本地伦理与法律资料在本阶段用于提供专业底线与记录规范，包括但不限于：

- 中国心理学会临床与咨询心理学工作伦理守则
- 中华人民共和国精神卫生法相关风险边界
- 个案记录与信息整理原则
- 风险识别、就医建议与转介边界
- 咨询关系、知情同意、保密与保密例外相关原则
- 对来访者福祉、专业胜任力和边界管理的要求
- 生物-心理-社会模型，用于来访综合身心状态评估和信息缺口检查

暂不在 MVP v0.1 中调用以下流派资料来改变信息收集模板：

- CBT 个案概念化框架
- 人本主义咨询框架
- 精神动力概念化框架
- 家庭系统理论框架
- 叙事、存在主义、焦点解决等流派框架

这些资料将在后续能力中使用，例如：

- 个案概念化
- 咨询方案设计
- 访谈问题提纲
- 咨询 Road Map
- 流派取向下的 session 反思与督导辅助

所有输出都应遵循：

- 区分“已知信息”“事实”“主诉”“初步假设”“推测”“待进一步评估”
- 信息不足时标记“待补充”，不得编造
- 不输出确定性诊断
- 不替咨询师做最终判断
- 涉及风险时提醒咨询师进行专业评估和必要转介
- 先给结果，再解释依据或参考资料来源

# 能力一：初访信息收集表生成

## 1. 用户意图

咨询师希望 Agent 生成一份统一的初访信息收集表，用于初访前预填写、初访过程中访谈，或作为系统字段配置。

该表单应服务于来访者基础信息和咨询相关信息的综合抓取，而不是根据咨询师流派生成不同理论版本。它需要为后续个案概念化、方案设计和报告生成提供足够稳定的原始资料。

## 2. 典型输入

- 给我一个初访信息收集表
- 生成一个来访者预填写版初访表
- 给我一个咨询师访谈用的初访提纲
- 帮我生成系统可落库的初访字段
- 生成一份专业通用的初访信息收集表

## 3. 输入结构

| 字段 | 是否必需 | 说明 |
|---|---|---|
| 表单用途 | 否 | 咨询师访谈版、来访者预填写版、系统字段版；未指定时默认咨询师访谈版 |
| 来访者类型 | 否 | 成人、青少年、儿童、伴侣、家庭等；未指定时默认成人个体咨询 |
| 咨询场景 | 否 | 个体咨询、学校咨询、机构咨询、线上咨询等 |
| 输出格式 | 否 | Markdown 表格、结构化列表、JSON Schema、前端表单配置等 |
| 特殊要求 | 否 | 例如更简洁、更温和、加入风险筛查、加入知情同意等 |

说明：本能力不接受“按 CBT 版本生成初访表”“按人本主义版本生成初访表”等流派化输出作为默认工作方式。如果用户提出类似需求，Agent 应说明初访信息抓取建议保持统一，并可在表单后附加少量“后续概念化可补充探索的问题”。

## 4. 输出结构

默认输出“咨询师访谈版”，结构包括：

1. 基本信息
2. 来访原因与咨询期待
3. 当前困扰评估
4. 重要生活背景
5. 既往心理与医疗相关信息
6. 风险评估
7. 个人资源与优势
8. 咨询目标初步设定
9. 咨询设置与知情同意确认
10. 咨询师初步记录

如用户要求来访者预填写版，应输出：

1. 填写说明
2. 基本信息
3. 这次想寻求的帮助
4. 最近的状态
5. 生活背景
6. 安全相关问题
7. 资源与期待
8. 咨询目标
9. 知情同意确认

如用户要求系统字段版，应输出：

1. schema_name
2. schema_version
3. sections
4. fields
5. field id
6. label
7. type
8. required
9. sensitive
10. risk_signal

## 5. 可能调用的 RAG 资料

- CPS 专业伦理与实践边界
- CPS 知情同意与保密原则
- 精神卫生法相关风险与就医建议边界
- 生物-心理-社会模型综合评估框架
- 初访评估规范
- 初访信息收集模板
- 机构初访表模板

## 6. 输出边界

- 不将表单写成诊断工具或某一流派的概念化工具
- 不暗示来访者必须回答所有问题
- 对风险问题保持清晰、谨慎和必要性说明
- 对来访者预填写版使用更温和、非评判的语言
- 系统字段版应标注敏感字段和风险信号字段
- 如咨询师要求按某一流派生成版本，应提醒流派框架更适合用于后续个案概念化，而不是替代基础信息收集模板

# 能力二：个案信息整理

## 1. 用户意图

咨询师希望 Agent 将个案背景、访谈笔记、初访记录或零散材料整理成结构化摘要，便于快速把握个案重点。

本能力应保持流派无关，重点是对来访信息进行综合抓取、分类和标注。它不直接套用 CBT、人本、精神动力等理论解释框架，而是为后续不同流派的个案概念化准备材料。

## 2. 典型输入

- 帮我整理这个来访者的背景信息
- 把这段笔录整理成结构化摘要
- 提取主要问题、关系线索和风险信号
- 帮我把这些零散记录整理一下
- 帮我整理这个个案信息

## 3. 输入结构

| 字段 | 是否必需 | 说明 |
|---|---|---|
| 个案原始材料 | 是 | 咨询师输入的个案背景、笔录、访谈记录或零散描述 |
| 整理目标 | 否 | 摘要、初访整理、风险线索提取、基础个案信息整理等 |
| 咨询阶段 | 否 | 初访、进行中、阶段总结、结案前等 |
| 咨询流派 | 否 | 可作为背景记录，但不改变本能力的基础整理结构 |
| 输出格式 | 否 | 结构化摘要、表格、报告段落等 |

## 4. 输出结构

Agent 应输出：

1. 主诉
2. 当前困扰
3. 关键事件
4. 情绪与认知线索
5. 行为模式
6. 人际关系线索
7. 资源与保护因素
8. 风险信号
9. 已知事实
10. 初步假设
11. 推测与待验证内容
12. 信息缺口
13. 建议咨询师进一步询问的问题

## 5. 可能调用的 RAG 资料

- CPS 个案记录与专业实践原则
- 精神卫生法相关风险与就医建议边界
- CPS 知情同意与保密原则
- 生物-心理-社会模型综合评估框架
- 初访评估规范
- 个案记录规范
- 风险评估规范
- CPS/中国本地伦理相关资料

## 6. 输出边界

- 必须区分事实与推测
- 不把推测写成事实
- 不扩写咨询师没有提供的个案信息
- 不输出确定性诊断
- 如果资料不足，应标记信息缺口
- 涉及自伤、自杀、他伤、虐待或现实检验问题时，应提示进一步专业评估
- 不在本能力中进行完整流派概念化；如用户要求按 CBT、人本、精神动力等流派解释，应转入后续“个案概念化”能力

# 能力三：Session 总结与咨询记录生成

## 1. 用户意图

咨询师希望 Agent 根据本次 session 的记录、口述反馈或笔记，生成结构化 session 总结、咨询记录和下次咨询重点。

本能力应优先完成专业记录和信息归档，保持流派无关。若咨询师希望进行 CBT、人本、精神动力等流派下的 session 反思，应作为后续扩展能力处理。

## 2. 典型输入

- 帮我总结今天这次 session
- 根据我的记录生成咨询记录
- 提取本次咨询重点和下次方向
- 帮我整理来访者这次的新信息
- 帮我生成本次咨询记录

## 3. 输入结构

| 字段 | 是否必需 | 说明 |
|---|---|---|
| Session 原始记录 | 是 | 咨询师输入的本次咨询笔记、转写文本或口述反馈 |
| 来访者基本背景 | 否 | 可选，用于帮助理解本次 session |
| 咨询阶段 | 否 | 初访、第几次咨询、阶段中、结案前等 |
| 记录格式 | 否 | 普通咨询记录、SOAP、DAP、BIRP、机构模板等 |
| 下次咨询需求 | 否 | 是否需要生成下次重点、访谈问题或家庭作业建议 |

## 4. 输出结构

Agent 应输出：

1. 本次主题
2. 来访者状态
3. 关键内容
4. 咨询师干预
5. 来访者反应
6. 风险变化
7. 进展与阻滞
8. 新增个案信息
9. 已知事实
10. 咨询师判断或初步假设
11. 待补充信息
12. 下次咨询重点
13. 可选的咨询记录正文

如果用户指定 SOAP 格式，应输出：

1. S：Subjective，来访者主观陈述
2. O：Objective，咨询师观察到的信息
3. A：Assessment，咨询师评估与初步理解
4. P：Plan，后续计划

如果用户指定 DAP 格式，应输出：

1. D：Data，本次资料与观察
2. A：Assessment，评估与理解
3. P：Plan，后续计划

如果用户指定 BIRP 格式，应输出：

1. B：Behavior，来访者表现与议题
2. I：Intervention，咨询师干预
3. R：Response，来访者反应
4. P：Plan，后续计划

## 5. 可能调用的 RAG 资料

- CPS 个案记录与专业实践原则
- CPS 保密与记录保存相关原则
- 精神卫生法相关风险与就医建议边界
- 咨询记录模板
- SOAP / DAP / BIRP 记录格式
- 机构记录规范

## 6. 输出边界

- 不添加咨询师未提供的信息
- 区分观察、来访者自述、咨询师判断
- 风险变化必须单独列出，即使结果是“未见新增风险信息”
- 如果原始记录中没有风险相关信息，应标记“未提供，建议咨询师按需评估”
- 不将 session 总结写成诊断结论
- 不替咨询师决定后续治疗方向，只提供可选建议
- 不在本能力中把 session 总结改写成某一流派的治疗解释；流派化理解应进入后续“个案概念化”或“方案设计”能力

# 4. 暂不纳入 MVP v0.1 的能力

以下能力暂不作为第一阶段核心功能，可放入 v0.2 或后续版本：

| 能力 | 暂缓原因 |
|---|---|
| 深度个案概念化 | 依赖流派框架、个案资料完整度和咨询师专业判断，建议在信息整理稳定后再做 |
| 咨询方案设计 | 涉及更高专业判断和风险管理，适合在 RAG 与边界机制更稳定后加入 |
| 咨询 Road Map | 需要结合咨询阶段、流派、目标和反馈动态调整，适合作为第二阶段能力 |
| 按流派切换信息收集模板 | v0.1 的目标是建立统一基础信息抓取系统；流派框架应在后续概念化与方案设计中使用 |
| 自动风险分级 | 风险评估高度敏感，第一阶段不做真实个案的最终自动分级或处置决策；训练/评测语境可输出候选危机等级及依据 |
| 直接面向来访者咨询 | 当前产品定位为咨询师端助理，不面向来访者提供咨询服务 |
| 自动诊断 | 不符合产品边界，不作为功能目标 |

## 5. MVP v0.1 成功标准

MVP v0.1 可视为完成，当 Agent 能稳定做到：

1. 根据简单指令生成初访信息收集表。
2. 根据咨询师输入的个案材料整理结构化摘要。
3. 根据 session 记录生成咨询记录和下次重点。
4. 三个 P0 能力都使用统一基础模板，不因咨询师流派不同而改变核心信息抓取结构。
5. 在所有输出中区分事实、主诉、推测、初步假设与待评估内容。
6. 信息不足时不编造，而是标记待补充。
7. 涉及风险内容时提醒咨询师进一步专业评估。
8. 能说明参考了通用咨询记录原则、伦理风险边界或相关 RAG 资料类别。

## 6. 下一步工作

完成本范围文档后，下一步可以继续拆解：

1. 为三个 P0 能力分别写 Agent 工作流。
2. 为每个能力设计标准提示词。
3. 为每个能力准备最小 RAG 资料清单。
4. 设计 3 到 5 个测试案例，验证输出是否稳定。
5. 将初访信息表和咨询记录结构转成前端表单或 JSON Schema。
6. 在 v0.2 中设计“个案概念化”能力，并允许咨询师选择 CBT、人本、精神动力、家庭系统等理论框架。
