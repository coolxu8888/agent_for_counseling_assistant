# 市场验证版关键安全决策

## 产品定位

市场验证版面向心理咨询师提供材料整理、结构化记录、概念化辅助、下次咨询计划和有限路线图草拟。系统不替代诊断、最终风险分级、危机处置或临床决策，所有输出必须由具备相应资质的专业人员复核。

## 数据边界

- 公开验证环境只允许使用去标识化材料。
- 不得录入姓名、电话、证件号码或其他可直接识别来访者的信息。
- API 密钥、上传材料、运行产物和工作区数据不得提交到 Git。
- 市场验证阶段的托管文件系统不能视为正式持久化存储；正式试点需要独立的数据保留、删除、备份和访问控制方案。

## 模型与 RAG 边界

- 中国本地法律、伦理和机构流程优先；境外专业来源只能作为补充，并保留地域适用性说明。
- RAG chunk 必须能够追溯到已审核来源卡；拒绝采用的来源也应保存拒绝理由。
- 模型必须区分已知事实、工作假设和待核实信息，不得把推断写成事实。
- 风险相关输出应保留材料依据、信息缺口和进一步人工评估提示，不自动作出处置决定。

## 验收与变更

- 能力完成以真实咨询师端到端路径为准：登录、建立去标识化个案、提交材料、审阅结果、生成可编辑文档、从历史记录重新打开并导出。
- 路由扩展只在真实用户路径被阻塞或误路由时进入优先级，避免用大量边界样例代替产品验收。
- 涉及 prompt、schema、RAG 来源、风险 rubric 或安全评分的变更，必须有对应回归测试或评测证据。
- 当前产品阶段统一称为“市场验证版”；历史文件中的 `v0.1` 仅用于标识当时的评测、范围或兼容契约。
