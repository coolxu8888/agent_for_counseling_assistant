# 市场验证版文档冻结设计

## 冻结结论

- 产品阶段统一命名为“市场验证版”（英文：`Market Validation Release`）。
- 不再使用 `v0.1`、`v0.2` 或 `MVP` 表达当前产品阶段。
- 现有 `0.1.0`、`v0.1` 文件名和 schema 字段仅在代码兼容性要求下保留，并明确标注为兼容标识，不代表产品版本。

## 清理目标

仓库中的 Markdown 只保留两类内容：

1. Web 上线、启动和运维实际需要执行的命令。
2. Web 运行时实际读取并拼装进模型上下文的 prompt 与 RAG 内容。

README 作为唯一人工入口，内容限制为版本声明、本地启动、Render 部署、必要环境变量、运行时 prompt 清单和兼容标识说明。

## 运行时白名单

保留并统一版本口径：

- `README.md`：精简后的唯一操作入口。
- `counselor-agent-v0.1-system-prompt.md`：代码直接读取的系统 prompt；文件名暂留以兼容现有路径。
- `rag/**/*.md`：Web 运行时检索和组装 prompt 的 RAG 内容；排除纯索引、评审指南、测试报告等未参与运行时检索的文件。
- `docs/coze-render-deployment.md`：仅当其命令或部署参数未能无损并入 README 时保留；优先合并后删除。

运行产物中的 `clean_output.md` 不属于仓库文档，不改变其生成逻辑。

## 删除范围

删除所有未被 Web 运行时读取、且其有效命令已并入 README 的 Markdown，包括：

- 旧版 MVP 范围、工作流、架构、能力地图和早期产品稿。
- 人工评测说明、历史评测结果和测试报告。
- `docs/superpowers/specs/` 与 `docs/superpowers/plans/` 下的历史设计和实施计划。
- 产品推进日志、集成计划及其他过程记录。
- RAG 目录中纯维护、索引、评审和历史测试文档。

实施前按代码读取关系生成最终白名单；任何被运行时代码实际读取的 Markdown 不删除。

## 文案规则

- 当前产品：`咨询师助理 Agent｜市场验证版`。
- 公开说明：仅使用去标识化演示材料，不录入真实可识别来访者信息。
- `v0.1` 若出现在兼容文件名、schema、评测产物名或固定路径中，附近必须注明“兼容标识”。
- 删除“后续 v0.2”路线图、已过期范围声明和研发过程叙述。

## 安全边界

- 不修改 Python、PowerShell、JavaScript、HTML、YAML 或 JSON 的运行行为。
- 不重命名被代码硬编码引用的 prompt/RAG 文件。
- 不覆盖当前工作区中已有的未提交改动；`docs/product-loop-state.md` 删除前先确认其改动已由现有工作保留在 Git 历史或另行处理。
- 不删除用户运行产物目录中的 Markdown。

## 验证

实施完成后验证：

1. 仓库剩余 Markdown 全部属于命令入口或运行时 prompt/RAG 白名单。
2. 面向产品阶段的文案只出现“市场验证版”。
3. 所有残留 `v0.1` 均属于兼容标识并有明确说明，或位于不可改名的机器契约中。
4. 本地 Web 启动命令和 Render `startCommand` 一致且可解析。
5. Web 工作台与 Agent Runner 的相关测试通过。
