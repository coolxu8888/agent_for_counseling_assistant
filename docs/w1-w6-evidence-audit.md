# W1-W6 Evidence Audit

Audit date: 2026-07-04

This is a conservative audit of the 30 workflow/gate cells required by the approved completion-matrix design. `passed` means the repository contains current, reproducible workflow-level proof. `unverified` does not mean the feature is absent; it means the available evidence does not prove this gate at the required scope. Historical prose and route-only checks are supporting context, not gate proof.

## Reproducible evidence used

- Local workflow suite (run during this audit): `$env:PYTHONPATH='scripts'; python -m unittest scripts.test_run_agent scripts.test_render_docx scripts.test_web_workbench` — 174 tests passed.
- Committed real-model evidence: `eval-results/eval-rubric-summary.v0.1.json`, with matching raw and cleaned artifacts under `eval-results/`, records passing DeepSeek results for W1 (`W1-001` through `W1-003`), W2 (`W2-001` through `W2-003`), and W3 (`W3-001`, `W3-003`).
- Web evidence: `scripts/test_web_workbench.py`, `scripts/web_workbench.py`, and `web-workbench/app.js`. W1 and W3 have workflow-specific API payload assertions and rendering code, but no test asserts actual browser rendering or export. API, code-presence, and routing assertions are explicitly classified as partial evidence rather than a passed Web gate.
- Hosted smoke capability: `scripts/hosted_smoke.py` and `scripts/run-hosted-smoke.ps1` provide a command mechanism, but no committed per-workflow successful smoke reports were found.
- Template evidence: `scripts/test_fill_docx_template.py`, `scripts/test_render_docx.py`, `eval-prompts/template-fill/`, and `docs/4.心理咨询初始访谈表_20210906.docx`. The committed `TF-001` fixture is XML rather than a real DOCX counselor template, and no committed successful W1-W6 real-template result report was found.

## Coverage summary and concerns

- All 30 cells are classified: 9 `passed`, 21 `unverified`, 0 `failed`.
- Passed by gate: local tests 6/6; real-model evaluation 3/6; Web integration 0/6; hosted verification 0/6; real-template verification 0/6.
- No workflow has all five gates passed; therefore no W1-W6 workflow is complete.
- `docs/product-loop-state.md` contains valuable historical claims about later W1-W6 live evals and hosted routing, but most corresponding result artifacts are not committed. Those claims cannot be reproduced or independently inspected from the repository.
- `scripts/hosted_smoke.py` can validate a hosted workflow, but a command's existence is not a successful run. Future evidence should commit a sanitized report containing the URL/version, workflow, command, timestamp, and asserted output fields.
- For W2, W4, W5, and W6, current Web tests establish routing or shared handler availability, not full workflow-specific presentation. Add an assertion from Web request through structured response and user-visible result/export for each workflow.
- A real-template gate should identify the real DOCX, exact workflow input/result, fill command, and committed sanitized fill report. Synthetic XML and in-memory DOCX fixtures are useful local tests but do not satisfy that gate.
