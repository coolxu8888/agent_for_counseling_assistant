# 咨询师助理 Agent 能力地图

## 1. Agent 角色定义

咨询师助理 Agent 是面向心理咨询师的专业辅助工具。

它不直接面向来访者提供心理咨询，不替代咨询师进行诊断或治疗决策。它的主要作用是帮助咨询师整理信息、生成文档、进行初步个案概念化、设计访谈提纲、规划咨询路线，并根据咨询师提供的反馈整理 session 内容与设计后续方案。

## 2. 核心使用对象

- 心理咨询师
- 心理治疗师
- 咨询机构督导或个案管理人员
- 心理服务平台的专业运营人员

## 3. 能力总览

| 能力模块 | 说明 |
|---|---|
| 信息收集表生成 | 生成初访信息收集表、来访者预填写表、系统字段表 |
| 个案信息整理 | 将咨询师输入的个案背景、访谈笔记、零散材料整理为结构化摘要 |
| 个案概念化 | 根据咨询师提供的信息，按指定流派生成个案概念化假设 |
| 访谈问题提纲 | 根据咨询目标、咨询阶段和咨询师流派生成访谈问题 |
| 咨询方案设计 | 为下一次或若干次咨询生成工作重点、目标和干预建议 |
| 咨询 Road Map | 生成阶段性咨询路线图，例如 4 次、8 次、12 次咨询计划 |
| Session 总结 | 根据咨询师反馈总结每次 session 的内容、进展、风险和下次重点 |
| 文档与报告生成 | 按模板生成咨询记录、初访评估、阶段总结、转介建议、结案报告 |
| 风险与伦理提醒 | 在识别高风险信息或伦理边界问题时提醒咨询师进一步评估 |

## 4. 用户意图分类

### 4.1 文档生成

**典型输入**

- 给我一个初访信息收集表
- 帮我生成一份咨询记录
- 根据这个模板生成阶段性报告
- 帮我写一个转介建议
- 帮我生成结案报告

**Agent 应输出**

- 结构化文档
- 可复制到工作系统中的文本
- 如有模板，严格按照模板格式输出
- 如信息不足，标记“待补充”，不要编造

**可能调用的 RAG 资料**

- 咨询记录模板
- 初访评估模板
- 阶段性报告模板
- 结案报告模板
- 机构书面材料规范
- 咨询伦理守则

**边界**

- 不虚构来访者信息
- 不替咨询师做最终诊断
- 涉及风险时只生成“风险提示”和“建议进一步评估”

---

### 4.2 个案信息整理

**典型输入**

- 帮我整理这个来访者的背景信息
- 把这段笔录整理成结构化摘要
- 提取主要问题、关系线索和风险信号
- 帮我把这些零散记录整理一下

**Agent 应输出**

- 主诉
- 当前困扰
- 关键事件
- 情绪与认知线索
- 行为模式
- 人际关系线索
- 资源与保护因素
- 风险信号
- 信息缺口

**可能调用的 RAG 资料**

- 初访评估规范
- 个案记录规范
- 风险评估规范
- CPS/中国本地伦理相关资料


**边界**

- 区分“已知事实”和“咨询师可进一步探索的假设”
- 不把推测写成事实
- 把推测和事实分开写

---

### 4.3 个案概念化

**典型输入**

- 根据 CBT 帮我做个案概念化
- 按精神动力取向整理这个个案
- 用家庭系统视角分析这个来访者
- 帮我形成一个初步概念化假设

**Agent 应输出**

- 核心困扰
- 触发因素
- 维持因素
- 保护因素
- 咨询目标假设
- 不同流派对应的理解框架
- 需要进一步验证的问题

**可能调用的 RAG 资料**

- CBT 个案概念化框架
- 精神动力概念化框架
- 家庭系统概念化框架
- 人本 / 存在 / 叙事等流派资料
- 咨询伦理守则

**边界**

- 使用“可能”“初步假设”“待进一步验证”等表述
- 不输出确定性诊断
- 不替代咨询师临床判断

---

### 4.4 访谈问题提纲

**典型输入**

- 下次咨询我该问什么？
- 按 CBT 给我设计 10 个访谈问题
- 我想探索他的亲密关系模式，帮我列提纲
- 根据这个个案帮我设计初访问题

**Agent 应输出**

- 访谈目标
- 问题分组
- 由浅入深的问题顺序
- 可选追问
- 咨询师注意事项
- 风险相关补充问题

**可能调用的 RAG 资料**

- 初访访谈框架
- 各流派访谈技术
- 风险评估问题库
- 咨询伦理守则

**边界**

- 问题应保持开放、非诱导、非评判
- 涉及创伤或高风险内容时提醒咨询师控制节奏

---

### 4.5 咨询方案设计

**典型输入**

- 这个个案下次咨询怎么推进？
- 根据来访者的情况给我一个咨询方案
- 他最近焦虑加重，我下次该怎么安排？
- 帮我制定接下来三次的咨询重点

**Agent 应输出**

- 当前阶段判断
- 本次 / 下次咨询目标
- 访谈重点
- 可选干预方向
- 家庭作业建议
- 风险管理提醒
- 咨询师可观察指标

**可能调用的 RAG 资料**

- 各流派干预原则
- 常用咨询技术库
- 风险评估规范
- 机构服务流程

**边界**

- 只提供建议，不替咨询师决定
- 不推荐超出咨询师资质范围的干预
- 对高风险个案提示转介或联合精神科评估

---

### 4.6 咨询 Road Map

**典型输入**

- 帮我做一个 8 次咨询计划
- 为这个个案设计阶段性咨询路线
- 按 CBT 做一个 12 次方案
- 我想给这个个案做短程咨询，帮我规划一下

**Agent 应输出**

- 咨询总目标
- 阶段划分
- 每阶段目标
- 每次 session 的重点
- 评估节点
- 可能风险与调整条件
- 结束或转介条件

**可能调用的 RAG 资料**

- 短程咨询框架
- CBT 结构化咨询框架
- 阶段性评估规范
- 结案与转介规范

**边界**

- Road Map 是可调整草案
- 需要根据来访者反馈和风险变化动态修订

---

### 4.7 Session 总结

**典型输入**

- 帮我总结今天这次 session
- 根据我的记录生成咨询记录
- 提取本次咨询重点和下次方向
- 帮我整理来访者这次的新信息

**Agent 应输出**

- 本次主题
- 来访者状态
- 关键内容
- 咨询师干预
- 来访者反应
- 风险变化
- 进展与阻滞
- 下次咨询重点
- 待补充信息

**可能调用的 RAG 资料**

- 咨询记录模板
- SOAP / DAP / BIRP 等记录格式
- 机构记录规范

**边界**

- 不添加咨询师未提供的信息
- 区分观察、来访者自述、咨询师判断

---

## 5. RAG 资料库分区

| RAG 分区 | 用途 |
|---|---|
| 伦理与风险库 | 保密原则、保密例外、危机干预、转介规范 |
| 初访评估库 | 初访问题、信息收集框架、风险筛查 |
| 流派框架库 | CBT、精神动力、家庭系统、人本、叙事等 |
| 咨询技术库 | 访谈技术、情绪调节、认知重构、关系探索等 |
| 文档模板库 | 咨询记录、初访评估、阶段报告、结案报告 |
| 机构规则库 | 服务流程、收费取消规则、记录保存规范 |

## 6. 通用输出原则

Agent 所有输出都应遵循：

- 使用专业、清晰、克制的语言
- 先给结果（方案、总结、答案），再解释或提供参考资料来源
- 明确标注“已知信息”“初步假设”“待进一步评估” “推测” “事实” “主诉” 
- 信息不足时提出补充问题
- 不编造个案信息
- 不输出确定性诊断
- 涉及风险时提醒咨询师进行专业评估和必要转介
