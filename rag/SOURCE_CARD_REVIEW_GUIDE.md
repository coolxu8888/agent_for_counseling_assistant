# Source Card Review Guide

本指南用于人工审核 `rag/source-cards/` 下的资料卡。资料卡审核通过后，才能切成正式 RAG chunk。

## 审核结论

每张资料卡只能处于以下状态之一：

| 状态 | 含义 | 下一步 |
|---|---|---|
| `approved` | 来源可靠，内容适合 v0.1 | 切成正式 RAG chunk |
| `pending` | 暂不确定，需要补充确认 | 保留资料卡，不入库 |
| `rejected` | 来源不适合或风险过高 | 记录拒绝原因，放入 rejected-sources |

## 审核清单

| 检查项 | 通过标准 |
|---|---|
| 来源可靠性 | 官方专业协会、政府/公共医疗机构、大学或可信机构 |
| 适用性 | 能服务初访、个案整理或 session 记录 |
| 地域语境 | 清楚标注 UK、US、全球或其他地区 |
| 伦理安全 | 不淡化自伤、自杀、他伤、虐待等高风险内容 |
| 专业边界 | 不鼓励 AI 替代咨询师判断 |
| 版权风险 | 不需要复制大段受版权保护内容 |
| 版本状态 | 资料没有明显过期；若可能过期，需标注 |
| 可切块性 | 能拆成 1 个或多个清晰 chunk |

## 审核记录格式

在资料卡 YAML 中更新：

```yaml
review_status: approved
reviewer: "审核人姓名或代号"
```

并在“人工审核备注”中补充：

```markdown
- 审核结论：
- 可切 chunk：
- 不应使用的部分：
- 适用限制：
```

## 常见处理

### 1. 来源可靠但地域限制明显

可以 `approved`，但正式 chunk 的“适用范围”必须写明地域限制。

### 2. 来源是第三方镜像

保持 `pending`，先找官方来源。

### 3. 来源可靠但内容太宽泛

可以 `approved`，只切与 v0.1 有关的小主题。

### 4. 涉及自杀量表或专有工具

除非确认授权，否则不要复制题项或评分规则。可以只保留“应关注的风险维度”。

### 5. 来源不适合

创建 rejected 记录，格式：

```markdown
---
source_id: ""
rejected_date: ""
reason: ""
---

# 拒绝原因

...
```
